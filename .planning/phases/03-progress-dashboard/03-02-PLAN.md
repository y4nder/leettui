<objective>
Deliver the first **visible, end-to-end** dashboard slice: a global `p` keybinding (browse AND problem view) opens a new full-screen `"dashboard"` view that reads the local submission store, computes the summary stats from plan 03-01's proven analytics engine, and renders them — current streak leading, then total + Easy/Medium/Hard breakdown, then 7d/30d counts + consistency % — in a scrollable, read-only viewport. `Esc`/`q` returns to the exact mode it was opened from (browse→browse, problem→that problem, state intact). An empty store shows a friendly backfill CTA.

After this slice a real user can press one key from anywhere and instantly read "am I improving and keeping it up?" from their real history — the core value of the phase. The heatmap and trend sparkline widgets are added in the next slice (03-03); the summary numbers are fully real here.

Purpose: Satisfies DASH-01 (new top-level view, global keybinding, `mode === "dashboard"`) and surfaces DASH-02/03/04/07's already-computed numbers to the user for the first time. Establishes the full render/route/bindings/return-mode plumbing that 03-03 extends.
Output: `DashboardView.tsx`, `dashboardSlice.ts`, the `"dashboard"` AppMode + return-mode wiring, the `dashboard.open`/`dashboard.close` commands + `p`/scroll/exit bindings, and the `app.tsx` route.
</objective>
