<objective>
Extend the now-visible dashboard with its two trajectory widgets: a **52-week × 7-day activity heatmap** whose per-cell color intensity encodes the number of first-time solves that day (DASH-05, D-12), and a **~12-week problems-per-week trend sparkline** rendered as Unicode block glyphs (DASH-06). Both are computed in plan 03-01's pure analytics module (extended here with `buildHeatmapGrid`/`buildWeeklyBuckets`/`buildSparkline`, unit-tested for correct indexing/scaling) and rendered in `DashboardView` below the summary block per the locked top-to-bottom order (summary → heatmap → trend, D-05).

After this slice the dashboard is feature-complete: a user reads not just "where am I" (the summary numbers) but "how has it been trending" — the consistency-over-time picture the phase exists to answer. Everything stays offline, uses OpenTUI primitives + Unicode glyphs, and adds no new dependency.

Purpose: Completes DASH-05 and DASH-06, closing the phase. Keeps the single counting unit (first-time solves, D-04) consistent across the whole screen — the heatmap and sparkline read the same first-AC set the summary does, so a re-solve-only day is blank on the heatmap and adds nothing to the trend.
Output: heatmap/sparkline functions in `analytics.ts` (+ tests), and the heatmap + sparkline render blocks in `DashboardView.tsx`.
</objective>
