<objective>
Build the phase's data + computation foundation as a **contract-first, unit-tested** engine: one new indexed read query (`getFirstAcSummary()`) that reduces the all-verdicts `submissions` table to one first-Accepted row per problem (joined to its difficulty), and a **pure, framework-free** analytics module (`src/ui/analytics.ts`) that turns those rows into every summary statistic the dashboard shows — total + Easy/Medium/Hard breakdown (DASH-02), current + longest streak (DASH-03), last-7/last-30 counts (DASH-04), and the rolling 30-day consistency score (DASH-07).

This is the MVP happy-path foundation: it ships as a TDD RED→GREEN slice (tests written first against the real computation contract), so plan 03-02's first visible slice consumes a proven, correct engine rather than untested aggregation glue. The one counting unit — a first-time distinct problem solve (D-01) — is locked into the code here, once, so every downstream number is coherent.

Purpose: A single trustworthy computation layer means every number on the dashboard is internally consistent (one DB snapshot, one reduction) and timezone-correct, and the tricky bits (millisecond unit, first-AC-once semantics, local-day bucketing) are pinned by tests before any UI exists.
Output: `getFirstAcSummary()` in `src/db/submissions.ts`; `src/ui/analytics.ts` + `src/ui/analytics.test.ts`.
</objective>
