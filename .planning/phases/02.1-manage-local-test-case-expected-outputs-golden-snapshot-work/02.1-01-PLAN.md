## Phase Goal

**As a** LeetCode solver working from the editor inner-loop (`:!leettui …`), **I want to** snapshot the current run's stdout into a case's expected output (`case-NN.out`) without hand-writing files, **so that** local `leettui test` grades real pass/fail instead of the verdict-less `ran`.

## Acceptance Criteria (phase-goal, this slice)
- [ ] `leettui test --save` in a problem's language folder writes a `.out` for each cleanly-run case and reports it.
- [ ] error/timeout cases are skipped (never blessed) and the skip is shown.
- [ ] a second `leettui test` grades the just-blessed case as pass.
- [ ] an unrelated case's `.out` is never touched.
