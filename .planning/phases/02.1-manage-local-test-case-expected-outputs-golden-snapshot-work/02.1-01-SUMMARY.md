# Phase 2.1 Plan 01: Golden-Snapshot `leettui test --save` Summary

**`leettui test --save` blesses the current offline run's stdout into `case-NN.out` via a new dir-injected `saveGoldenOutputs` writer, closing the gap where seeded test cases could only ever grade as the verdict-less `ran`.**

## Accomplishments

- New core writer `saveGoldenOutputs(testsDir, cases)` blesses every cleanly-run `LocalCaseResult` into a sibling `case-NN.out`, reusing `testRunner.compareOutput`'s JSON-normalized grading to decide `created`/`changed`/`unchanged`, skipping `error`/`timeout` cases (D-02) and never writing outside `join(testsDir, "{name}.out")` (D-04).
- `cli/present.ts` gained a full save-reporting family — `saveOutcomeGlyph` (private), `presentSaveSummary`, `saveReminder`, `exitCodeForSave` — reusing the existing TTY/`NO_COLOR`-gated palette and the `exitCodeForLocalRun` 0/2 rationale.
- `leettui test --save` is now live: `runCli`'s third parameter is the full `argv` (was a single optional `arg`), with new `hasFlag`/`flagValue` helpers parsing `--save` (and, ready for plan 02, `--add-case`'s file argument). The `test` branch runs `runLocalTests` exactly once (D-12 — no re-run/re-compile) and only blesses the `ran` arm; every other report kind (`unsupported`/`no-harness`/`no-cases`/`compile-error`) behaves byte-for-byte like plain `test`.
- TCASE-01/02/03 formalized in `.planning/REQUIREMENTS.md` under a new "Test-Case Management (Phase 2.1)" subsection, with traceability rows and a bumped coverage total (18 → 21).

## Decisions Made

- `saveGoldenOutputs` is dir-injected (`testsDir: string`, not `(id, titleSlug)`) so the writer is unit-testable in-process against a real temp dir, matching `discoverCases`'s existing shape — the CLI resolves `getTestsDir(id, slug)` and passes it in.
- Used synchronous `node:fs` `readFileSync`/`writeFileSync` rather than `Bun.write` (which returns an unawaited promise in the `seedTests` idiom) — the create-vs-overwrite decision and the write both need to complete within one synchronous call so in-process tests can `readFileSync` immediately afterward with no race.
- `saveOutcomeGlyph` stayed a private, non-exported helper (mirroring `caseGlyph`) since nothing needs to unit-test the glyph mapping directly — `presentSaveSummary`'s rendered output is what's asserted.
