### `src/core/solutions/save-output.ts` (new helper — `.out`-writer)

**Analog:** `src/core/solutions/create.ts` → `seedTests` (lines 89-104)

**Imports pattern** (mirror `create.ts` lines 1-15):
```typescript
import { existsSync, mkdirSync } from "node:fs";
import { dirname, join } from "node:path";
import { getTestsDir } from "./paths";
```

**Core create-if-absent/overwrite idiom to mirror** (`seedTests`, lines 93-104):
```typescript
export function seedTests(id: number, titleSlug: string, exampleTestcases: string[]): void {
  if (!exampleTestcases || exampleTestcases.length === 0) return;
  const dir = getTestsDir(id, titleSlug);
  mkdirSync(dir, { recursive: true });
  exampleTestcases.forEach((testcase, i) => {
    const name = `case-${String(i + 1).padStart(2, "0")}.txt`;
    const path = join(dir, name);
    if (!existsSync(path)) {
      Bun.write(path, testcase);
    }
  });
}
```
**Key divergence from `seedTests` (per D-03/D-04):** the new `.out`-writer is
**overwrite-if-exists, not create-if-absent** — `seedTests` skips an existing
file; `saveOutput`/`saveGoldenOutputs` must intentionally **overwrite** the same
case's `.out` (golden-update / `jest -u` model) while still creating the dir
with `mkdirSync(..., { recursive: true })`. Suggested per-case-outcome shape,
consumed straight from `LocalRunReport.ran.cases` (`testRunner.ts` lines 47-52,
64):
```typescript
import type { LocalCaseResult } from "../testRunner";

export type SaveOutcome = "created" | "changed" | "unchanged" | "skipped";

export interface SaveResult {
  name: string;
  outcome: SaveOutcome;
}

// Never-throws / best-effort, like seedTests and git.ts. Only cases with
// status "ran" or "pass" are blessed (D-02); "fail" is also acceptable to
// re-bless per D-03 (overwrite), "error"/"timeout" are always skipped.
export function saveGoldenOutputs(
  id: number,
  titleSlug: string,
  cases: LocalCaseResult[],
): SaveResult[] {
  const dir = getTestsDir(id, titleSlug);
  return cases.map((c): SaveResult => {
    if (c.status === "error" || c.status === "timeout" || c.actual === undefined) {
      return { name: c.name, outcome: "skipped" };
    }
    mkdirSync(dir, { recursive: true });
    const path = join(dir, `${c.name}.out`);
    const existed = existsSync(path);
    const prev = existed ? Bun.file(path).text() : null; // (await if made async)
    Bun.write(path, c.actual);
    if (!existed) return { name: c.name, outcome: "created" };
    // compare prev vs c.actual to decide "changed" vs "unchanged" — reuse
    // testRunner.ts's normalize()/compareOutput-style trimEnd comparison, or a
    // simple string equality since `.actual` is already trimEnd()'d upstream.
    return { name: c.name, outcome: "changed" }; // or "unchanged" — planner's call
  });
}
```

### `src/core/solutions/add-case.ts` (new helper — `--add-case` input writer)

**Analog:** `src/core/solutions/create.ts` → `seedTests` (numbering/padding convention, lines 96-100) + `discoverCases`/`pairCases` in `src/core/testRunner.ts` (lines 75-142, for computing the next index)

**Core pattern to mirror** — reuse the exact `case-${NN}` zero-pad convention
from `seedTests` (`String(i + 1).padStart(2, "0")`), but compute the **next**
index instead of a fixed loop index, via `discoverCases`/`pairCases`:
```typescript
import { mkdirSync } from "node:fs";
import { join } from "node:path";
import { getTestsDir } from "./paths";
import { discoverCases } from "../testRunner";

// D-09 (Claude's discretion): next sequential index, zero-padded to match
// seedTests' width (2 digits, "case-01".."case-99"; consider widening if >99
// cases is plausible — planner's call).
export function nextCaseName(id: number, titleSlug: string): string {
  const existing = discoverCases(getTestsDir(id, titleSlug));
  const nums = existing.map((c) => parseInt(c.name.replace("case-", ""), 10));
  const next = nums.length ? Math.max(...nums) + 1 : 1;
  return `case-${String(next).padStart(2, "0")}`;
}

// Writes the new case's INPUT only (never an .out) — mirrors seedTests' single
// Bun.write, always creates (never collides since nextCaseName is derived from
// discoverCases' current state).
export function addCase(id: number, titleSlug: string, input: string): string {
  const dir = getTestsDir(id, titleSlug);
  mkdirSync(dir, { recursive: true });
  const name = nextCaseName(id, titleSlug);
  const path = join(dir, `${name}.txt`);
  Bun.write(path, input);
  return path;
}
```
Note: `discoverCases` is currently exported from `testRunner.ts`, not
`solutions/`; either import it directly from `../testRunner` (as above) or have
the planner promote it into a shared location — no existing barrel re-exports it
from `core/solutions`, so this is a cross-module import, consistent with how
`create.ts` already imports from `../harness`.

**stdin/file input reading** — no existing analog for stdin reads in the
codebase; use Bun's own idiom (`await Bun.stdin.text()` when no file arg, else
`await Bun.file(path).text()`), applied in `src/cli/index.ts`'s new flag branch,
not inside the core helper (keep `addCase` pure/fs-only, matching `seedTests`
taking already-resolved strings rather than reading stdin itself).
