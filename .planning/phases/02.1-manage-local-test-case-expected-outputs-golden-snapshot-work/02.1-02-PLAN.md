<objective>
Deliver the second capability of Phase 2.1: `leettui test --add-case`, which lets the user add a brand-new local test case without hand-creating files. It reads the case INPUT from stdin or a file argument (D-05), writes it as the next sequential `case-NN.txt` in the problem's `tests/` dir, and stops there — the new case's expected output is blessed by a follow-up `leettui test --save` (the two-step flow, D-05). The written input is picked up by the existing `discoverCases`/`pairCases` on the next run with zero runner changes (D-06).

This slice depends on plan 01: it shares `src/cli/index.ts` (the argv-threaded `runCli` + the `hasFlag`/`flagValue` helpers) and `src/core/solutions/index.ts` (the barrel), and the add-case → save → grade round-trip is only meaningful once `--save` exists — hence wave 2.

| stdin / file argument → `case-NN.txt` on disk | The new case input is read from stdin or a user-named file and written verbatim into the problem's `tests/` dir. |
| editor `:!` (non-TTY child) → `leettui test --add-case` | Runs non-interactively from `$EDITOR`; input is piped, no prompt. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02.1-04 | Tampering | addCase write path | high | mitigate | The written path is `join(testsDir, `${nextCaseName(testsDir)}.txt`)`; the filename comes from the internal `case-NN` counter (max existing + 1), never from external input, so a new case can never overwrite or collide with an existing one (D-04/D-09). Unit-tested for the gap/monotonic cases. |
| T-02.1-05 | Denial of Service | empty / unreadable input | low | mitigate | Empty/whitespace input exits 2 without writing (never a blank case); a missing file surfaces via `Bun.file(...).text()` rejecting — caught by the verb's normal error path, exit 2 (couldn't-run family), consistent with cli/CLAUDE.md. |
| T-02.1-06 | Information Disclosure | reading an attacker-named file | low | accept | `--add-case <file>` reads exactly the path the user typed from their own editor; it only copies content into the user's own solutions tree, no privileged read or exfiltration. |

Note: this plan installs NO new packages (only `node:fs`, `node:path`, Bun built-ins already in use) — no package-legitimacy checkpoint or `T-*-SC` supply-chain threat applies.
