<objective>
Deliver the browse attempt-count badge (BROWSE-01): a raw `×N` count shown in the browse question list on problems the user has submitted before, richer than LeetCode's binary solved/attempted mark. The badge reuses the existing `◆` solution-file-marker slot and takes precedence over it (D-13/D-14), reads a new aggregate SQL count, and refreshes wherever the `◆` marker already refreshes — plus the backfill-completion path, which today refreshes nothing.

Purpose: Turn Phase 1's stored submission history into an at-a-glance "how much have I worked this one?" signal in the always-visible browse list — no new schema (the new GROUP BY query hits the existing `idx_sub_question_time` index), no new dependencies.
Output: a new `getSubmissionCountsByQuestion()` query, an `attemptCounts` Map on `questionsSlice`, the badge render in `QuestionList`, and the backfill-refresh fix.
</objective>

## Phase Goal

**As a** LeetCode grinder browsing the problem list, **I want to** see an attempt-count badge on problems I've submitted before, **so that** I can tell at a glance which ones I've already worked on and how much.
