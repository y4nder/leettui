<objective>
Deliver the per-problem submission-history panel in ProblemView (HIST-01, HIST-02, HIST-03). Opening a problem surfaces every stored attempt (any verdict) newest-first as a compact, color-coded single-line row, with a best-vs-latest AC runtime summary and subtle inline percentile. The panel is the 4th focusable right-column panel, navigate-only (Enter reserved for a future submission-detail view, D-10).

Purpose: Turn the submission data Phase 1 already stores into an at-a-glance "am I improving on this problem?" view exactly where the user solves — no new API calls, no new schema, pure read + derive + render over `getSubmissionsForQuestion()`.
Output: `src/ui/verdict.ts` (+ test), `src/ui/components/HistoryPanel.tsx`, `src/ui/store/slices/submissionsSlice.ts`, and extensions to `problemSlice`, the problem handlers, ProblemView, and the keymap.
</objective>

## Phase Goal

**As a** LeetCode grinder working a problem in the terminal, **I want to** see my full attempt history — verdicts, runtimes, and whether my latest solve beat my best — the moment I open the problem, **so that** I can tell at a glance whether a re-solve actually improved.
