<objective>
Lay the irreducible storage foundation for the submission-history milestone: add a `submissions` table (keyed by LeetCode `submissionId` PRIMARY KEY) plus a `submissions_fetched_at` cursor column on `questions`, both inside the existing `questions.db`; ship them as a generated-and-embedded Drizzle migration; and provide the `submissions.ts` CRUD module every later plan/phase reads.

Purpose: Nothing in this milestone can store or read submission history until this table exists. Keying on `submissionId` (DATA-01) makes every write idempotent and lets the existing `*.db` gitignore protection cover history automatically (ROADMAP success criterion 5). The free-text `statusDisplay` column stores all verdicts (D-08), the superset Phase 2's per-problem panel and Phase 3's dashboard depend on. The `submissions_fetched_at` column is the single field that serves both cursor roles (D-07): a resume marker for an interrupted run and a high-water mark for a completed import.

Output: A migrated `submissions` table + `submissions_fetched_at` column, an embedded `0002` migration, and a tested `src/db/submissions.ts` CRUD module.
</objective>
