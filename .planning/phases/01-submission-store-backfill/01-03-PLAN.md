## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LeetCode REST CheckResponse → leettui | Untrusted submit-result fields mapped into a stored row |
| UI command → core backfill service | A user action starts/cancels a long-running async task |
| `session.json` (cross-boot state) → boot logic | The one-time nudge flag persists between runs |
| `questions.db` → user's git/solutions repo | Submission history must stay behind the `*.db` gitignore |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-CHECKINPUT | Tampering | CheckResponse → DbSubmission append | medium | mitigate | Nullable guards + `parseInt` on every CheckResponse field; insert only when `submission_id` is present; `Date.now()` for the timestamp — V5 Input Validation |
| T-01-FREEZE | Denial of Service | progress wiring | medium | mitigate | The service runs as a plain `async` task (event loop runs between awaits) so the render loop never freezes; `clearSyncProgress()` in a `finally` guarantees the bar never sticks (DATA-07, Pitfall 6) |
| T-01-CANCEL | Denial of Service | long-running backfill | low | mitigate | `CancelRef` lets the user stop a runaway/over-long run; partial data preserved; cancel is a no-op when idle (D-05) |
| T-01-NUDGESPAM | Annoyance/Integrity | first-run nudge | low | mitigate | `setBackfillNudgeShown()` called unconditionally + suppressed when history exists → fires at most once, never nags (D-01/D-02, consistent with the no-nag stance) |
| T-01-DISCLOSE | Information Disclosure | storage location | high | mitigate | Appended rows land in `questions.db`, already covered by the `*.db` gitignore — verified in the human checkpoint step 6 (ROADMAP success criterion 5) |
| T-01-SC | Supply Chain | dependencies | low | accept | No new packages introduced (RESEARCH §Package Legitimacy Audit: none) |
</threat_model>
