<objective>
Build the API and core-service half of the backfill: a `submissionList` GraphQL query module and a never-throws, resumable, polite `backfillSubmissions` service that paginates the user's history into the `submissions` table created in 01-01. Critically, a live spike validates the reverse-engineered (and therefore LOW-confidence) LeetCode API shape and the real rate-limit behavior BEFORE the query field set and the inter-page delay are locked.

Purpose: DATA-02 (paginated `submissionList` capturing status/lang/runtime/memory/timestamp per attempt), DATA-03 (resumable + idempotent ‚Äî persisted cursor + `onConflictDoNothing`), DATA-04 (polite ‚Äî configurable inter-page delay + 429 backoff, on-demand only), and DATA-05 (never crashes ‚Äî discriminated `BackfillResult`, preserves partial data, mirrors `core/git.ts`). The spike de-risks the API assumptions flagged throughout RESEARCH.md (A1-A9) so the locked defaults are observed, not guessed.

Output: `scripts/spike-submission-list.ts` (throwaway probe + recorded findings), `src/api/queries/submission-list.ts`, `src/core/backfill.ts`, and `src/core/backfill.test.ts`.
</objective>
