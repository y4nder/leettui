## Phase Goal

**As a** LeetCode solver working from the terminal, **I want to** have leettui automatically
save a failing run/submit's input and expected output as an offline test case, **so that** I
can reproduce and fix the bug locally with `leettui test` without repeatedly hitting LeetCode.

## Artifacts this phase produces

New symbols introduced by this plan (source-grounding should treat these as newly-created):

- `notes?: string[]` — new optional field on `ResultView` (`src/views/browse/resultView/types.ts`)
- `captureFromSubmitResult(question, langSlug, result)` — internal branching helper in `src/core/submission.ts` (naming at planner discretion; D-08)
- `captureFromRunResult(question, langSlug, exampleTestcaseList, result)` — internal branching helper in `src/core/submission.ts`
- Return-shape change: `runSolution` / `submitSolution` now return `{ response: ParsedResponse; captureNotes: string[] }`

(`captureFailingCase`, `blessRunOutputs`, `CaptureResult`, `normalize` are produced by Plan 01 — not new here.)
